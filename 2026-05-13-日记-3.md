## 粉白几何风的三次迭代

主人一开始说"变漂亮一点"，后来提了具体方向：粉白配色 + 神秘几何切割感。

**第一版（翻车）：** 惠惠理解成全界面几何风——把 `rounded_corners_ui` 关了（全直角），给滚动条加了 `clip-path: polygon(...)`，文章卡片用了对角渐变遮罩。粉色用了樱花粉 `#EC407A`。结果主人说樱花粉太扎眼、圆角要留着、切割感指的是首页头图不是整个界面 (;´Д`)

**第二版（修正）：** 粉色调柔化为灰调玫瑰 `#D4A0A8`（dusty rose），圆角恢复 `true`，几何切割只作用于 `#page-header`。头图 CSS 用了三层 `linear-gradient` 对角斜切 + 两个 `::before` / `::after` 伪元素配合 `clip-path: polygon()` 做 diamond 裁切，透明度都控制在 0.04~0.12 之间，低调不抢眼。

配色细节全改了：
- main: `#D4A0A8`（灰调玫瑰）
- paginator: `#E0B8C0`
- link_color: `#C89098`
- code_background: `rgba(240, 210, 218, 0.3)`
- blockquote_background_color: `#F8E8EC`
- meta_theme_color_light: `fff8f9`

分割线图标设了 Font Awesome 钻石 `\f3a5`（fa-gem），和几何切割呼应。

**头像：** 主人放了 `/img/head.jpg`，改了 `avatar.img` 配置，顺便 social 链接里的 GitHub 从占位符改成了 `https://github.com/Yenmor`。

## 便捷脚本的一波三折

主人说想要预设脚本用来创建文章和推送。

**第一版：** 放在根目录 `yenmorBlog/`，用命令行参数 `./new-post.sh "标题"`。三个文件：new-post.sh、push.sh、preview.sh。

**第二版：** 主人说要放进 `scripts/` 文件夹，要有 sh 和 bat 双版本，参数改成运行时输入（`read` / `set /p`）。惠惠照做了，还加了中文提示。

**第三版（CMD 编码炸了）：** 主人运行 `push.bat` 报错——中文 + emoji + `&&` 链全变成乱码，CMD 把乱码当命令执行了。`chcp 65001` 也不管用。最后主人说"我看得懂英文"，惠惠顿悟——纯英文 ASCII 重写所有 `.bat`，去掉 emoji，`&&` 链拆成逐行 + `if %errorlevel% neq 0` 检查。瞬间好了。

**第四版（Hexo 脚本目录冲突）：** 安装 `hexo-abbrlink` 后 `hexo generate` 报错——Hexo 把项目根目录的 `scripts/` 当作自己的 JS 插件目录，尝试把 `.bat` 当 Node.js 代码加载！`@echo off` 被当 JS 语法错误了。解决方案：整个文件夹改名 `tools/`，Hexo 就不碰了。但旧 `scripts/` 因为主人终端还停留在里面删不掉，暂时留着。

**第五版（git add . 范围 bug）：** 主人运行 `tools/push.bat` 提交，结果 `git add .` 只在 `tools/` 目录内生效——项目外面的文章改动没被 stage。原因是从其他地方运行脚本时当前目录不对。修复：所有脚本开头加了 `cd` 到项目根目录——bash 用 `cd "$(dirname "$0")/.."`，bat 用 `cd /d "%~dp0.."`。

## 短链接 hexo-abbrlink

主人说文章链接里中文被 URL 编码成一长串 `%E4%BD%BF%E7%94%A8...` 太丑。备选方案是拼音（hexo-permalink-pinyin）和短 ID（hexo-abbrlink），主人选了短 ID。

安装 `hexo-abbrlink`，在 `_config.yml` 里配置：
```yaml
permalink: posts/:abbrlink.html
abbrlink:
  alg: crc32
  rep: hex
```

效果：`/2024/11/24/使用PicGo + 腾讯云COS搭建md图床/` → `/posts/9f907cb9.html`，干净多了。
